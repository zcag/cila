# Brief — App dashboard (logged-in view)

**Type:** application dashboard (in-product screen) · **Stack:** Next.js

Build the main dashboard for **Cadence**, a tool sales teams use to track deal momentum. This is the screen a rep sees after login — information-dense but not cluttered. It needs: a top summary band (pipeline value, deals closing this week, momentum trend), a primary table/list of active deals (company, stage, value, last touch, a momentum indicator), a focus panel for the deal that needs attention today, and a clear primary action ("Log activity"). Use realistic seeded data, not "Lorem deal." Must handle a loading and an empty state. Dashboards are where AI output goes generic-shadcn fastest — give it real hierarchy, restraint, and a point of view; legibility and scannability beat decoration.
