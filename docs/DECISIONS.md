# cila — Decisions

Locked decisions and rationale. Append-only; supersede with a dated note rather than deleting.

- **[name]** Called **cila** (Turkish: *polish/gloss*) — short, types well as a command prefix (`/cila:design`), names exactly what it does (the craft/polish layer over raw components).
- **[form]** A **Claude Code plugin**, not a starter template — the only container that bundles commands + subagents + skills + hooks + MCP config as one installable, portable unit. Satisfies the "point any agent at it from anywhere" requirement. Per-repo template lives in `templates/` and is materialized by `/cila:init`.
- **[approach]** **Composition, not from-scratch.** ~80% existing OSS (frontend-design, OneRedOak design-review, Vercel guidelines, accessibility-agents, shadcn MCP, Playwright MCP, design-extract). cila adds only the ~20% missing: collaborative design step, locked token contract, per-repo materialization, glue. Depend on `frontend-design` via `plugin.json` `dependencies` — do not fork it.
- **[cost]** **100% free stack by default.** Component *components* (Aceternity/Magic UI/Origin/Cult/etc.) are free/MIT; only their **Pro template bundles** (~$199) are paid and never required. Chromatic and Figma seats are optional. The one real recurring cost is Claude plan quota (visual loop ≈ 3x vision tokens) — mitigated by capping screenshots/cycle and on-demand review.
- **[location]** Authored in `~/proj/cila` (matches the `~/proj/claude-skills` pattern); distributed via a local marketplace synced through `~/dotty` so the fleet (marko/bender/archer) gets it. GitHub repo **private**.
- **[stacks]** Astro + Tailwind v4 + Cloudflare Pages for marketing (Astro-first); Next.js + Tailwind v4 + Vercel for apps. Tokens OKLCH via tweakcn; fonts self-hosted via Fontsource defaulting to Fontshare faces; motion via `motion/react` (not Webflow-owned GSAP for generated output).
- **[phasing]** Ship **Phase 1 (lean core)** first; Phases 2 (compounding) and 3 (scale & wow) are opt-in upgrades. Keep Phase 1 short/dense/readable — no over-engineering.
- **[gates]** Acceptance = green hard gates + screen-reader spot-check, **never** "the screenshot looks good." LLM/perceptual judge is advisory and runs only after structural gates pass. Build to WCAG 2.2 AA (3.0 is a draft, ~2029).
- **[review-loop]** The **generator never grades its own output** — a separate evaluator subagent does, with Playwright MCP isolated in its own context window.
- **[wow]** **Jaw-dropping is a goal, not an afterthought.** cila chases award-tier hero moments (3D/shaders/generative backgrounds/orchestrated motion), anchored on real references (via the Steel MCP) and committed to ONE signature moment. Hard rule: spectacle never trades away **accessibility or reduced-motion** — those gates apply to showcase pages too. **Performance** flexes via an opt-in **showcase Lighthouse profile** (`lighthouserc.showcase.json`), while CLS + compositor-only animation stay strict. Wow uses **vetted, build-verified patterns** (`templates/wow/`, `skills/wow|r3f|shaders`) because agents fail at freehand WebGL. Tier the effect: CSS/SVG first, R3F/shaders when depth is needed.
- **[ux]** **One front door, no exposed internals.** The user interacts through a single model-invocable skill **`/cila:go`** (which also auto-fires when they describe a site to build). cila **auto-detects all mechanics** — new vs existing repo, scaffold vs adopt/overlay, framework, which gates — with **no flags or modes**. It collaborates with the user *only* on the visual direction and "does this look right"; everything technical is silent, and no jargon ("adopt", "gates", "subagents", "tokens") is surfaced. The complementary principle: auto-handle mechanics, but never assume *intent/taste* or overwrite work without surfacing it. *Supersedes the earlier per-command UX — `/cila:design|init|review` removed; their logic now lives in the `go` orchestrator + the subagents.*
- **[orchestration]** `/cila:go` is a **staged contract**, not loose steps: ordered stages (assess → direction → materialize → build → review → done), each with an **exit criterion**, plus two standing rules — *never skip a stage; never declare done with a failing gate* — and a `.cila/state.json` progress flag. A **`Stop`/`SubagentStop`-gate hook** (`hooks/stop-gate.sh`) reads that flag and blocks completion while there are ungated UI changes. **Honest limit (chosen):** the flag is model-written, so the hook is a strong *reminder/nudge*, not un-bypassable enforcement — real enforcement is the gates actually running in Stage 4 (+ CI if wired). The hook is cheap (file check), scoped (no-op unless `gate_required:true`), and loop-safe (`stop_hook_active`). Hardening it (run `gate:structural` in-hook, or a verified `gate_pass_evidence` field) is a future option. A full per-stage-subagent harness / `features.json` stays Phase 3.
- **[integrations]** External integrations needing a key/account/paid plan (AI image-gen, Mobbin, poly.pizza, Figma MCP, v0 API, Chrome DevTools) are **opt-in and documented in `docs/INTEGRATIONS.md`** — never hard-wired into the plugin `.mcp.json`, so nothing breaks without setup. The default stack stays free/keyless (shadcn + Playwright MCP; Iconify/Poly Haven/Steel/Fontsource used directly).
- **[cut] Headless `cila-run` + CI/PR automation** was removed at the user's request — cila is an interactive design partner, not an unattended runner.
- **[content]** **Content is a first-class layer**, not an afterthought — web design = content + visual + production, and cila must wield all three. A `CONTENT.md` contract (positioning · ONE core message · voice chart · page/section plan · VoC) parallels `DESIGN.md`, produced by the **content-strategist** subagent + the `copywriting` / `voice` / `content-structure` skills. **Content-first:** the message is decided before/with the look, and content hierarchy drives visual hierarchy (build writes real copy, never lorem). The **anti-slop kill-list** is enforced and `design-reviewer` critiques the copy (5-second clarity, "So what?", scannability). Evidence: `docs/RESEARCH-CONTENT.md`.
- **[scope] Build-time only — cila does not care about deployment or deployed state.** No analytics instrumentation in shipped sites, no RUM, no live user experiments (A/B / multivariate / CRO), no conversion-feedback loops. Deployment is the user's concern. cila's quality signal is **build-time** (the gates + a dev-time benchmark) and any self-improvement comes from the **user's edits + that benchmark**, never production telemetry. *(Cuts the production-feedback parts of the Theme C frontier research; build-time scaling — programmatic pages, design-system governance, i18n, redesign-audit — stays in scope. CWV/INP gates run at build/CI against a local server, not production monitoring, so they're fine.)*
