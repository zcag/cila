# cila — Content & Marketing Research Archive

Evidence base for cila's **content layer** (the third leg of web design: content + visual + production). Four deep-research passes, distilled. Frameworks named; sources at the end of each section. Pairs with [`RESEARCH.md`](RESEARCH.md) (visual/production).

The whole layer composes into one chain:
**JTBD job/struggle → VPC ranked pains/gains → Dunford positioning → ONE core message + pillars → Schwartz awareness (where the page leads) → VoC mining (the actual words) → StoryBrand arc + IA → conversion-structured, voice-driven, anti-slop copy → content-first handoff to the visual layer.**

## 1. Positioning & messaging strategy (what to say)

**April Dunford — positioning is discovered, not declared.** Five interdependent components, reasoned as a chain: **competitive alternatives** (incl. "do nothing / spreadsheet / hire an intern") → **unique attributes** (with proof) → **value themes** (cluster attributes into 3–4) → **who cares a lot** (the ICP, defined by what makes them care) → **market category** (the frame that makes the value obvious). Optional **trend** for "why now". Three styles: head-to-head · big-fish-small-pond · create-a-new-category. Reject the fill-in-the-blank positioning statement as a *starting* point — use it only to *capture* output.

**Value Proposition Canvas (Strategyzer).** Customer profile = jobs (functional/social/emotional) · pains · gains; value map = products · pain relievers · gain creators. **Rank** jobs/pains/gains and address only the top few — the #1 mistake is trying to relieve every pain. Capture the prop with Moore (*For [customer] who [need], [product] is a [category] that [benefit]. Unlike [alternative], we [differentiation].*) or Blank (*We help X do Y by Z*).

**Jobs-To-Be-Done.** Customers "hire" a product to make **progress** in a **circumstance**. Switch happens only when **Push + Pull > Habit + Anxiety** — messaging amplifies push/pull, reduces anxiety (proof/guarantees/ease), breaks habit. Job story: *"When [situation], I want to [motivation], so I can [outcome]."* The **struggling moment** is the richest messaging source; interview recent switchers.

**Eugene Schwartz — 5 stages of awareness** set where the page leads: Unaware (story/problem) · Problem-aware (lead with the problem) · Solution-aware (lead with the outcome/approach) · Product-aware (lead with the product + proof) · Most-aware (lead with the offer). Match headline + lead to the dominant stage of best-fit visitors.

**Message hierarchy:** ONE core message (the spear tip → hero headline) + 3–4 supporting pillars (each = a value theme + unique attribute + proof → a page section) + a proof layer + an explicit **cut list** (parity claims demoted to table stakes). Differentiation only exists relative to the named alternative; run the **"who else could say this?"** parity test.

**Voice-of-Customer mining (Wiebe).** The best copy is *found, not written* — swipe sticky phrasing from reviews/interviews/support/search. (Wiebe's classic: a mined line beat invented copy +400% CTA clicks.) Mine pains, gains, before/after, objections, triggers, verbatim phrases. Review hack: `site:amazon.com inurl:"product-reviews" "tired of" [keyword]`.

Failure modes to guard: feature dumps (not value themes), parity claims as differentiation, demographic ICPs (not job/circumstance), ignoring "do nothing", inventing copy instead of mining.

## 2. Conversion copywriting & page structure (how to phrase + sequence)

**Frameworks as section-level scaffolds** (pick per awareness stage): **PAS** (Problem-Agitate-Solution — problem section, cold traffic) · **AIDA** (whole-page arc) · **BAB** (Before-After-Bridge — aspirational hero) · **FAB** (Feature→Advantage→Benefit — feature sections) · **4 U's** (Useful/Urgent/Unique/Ultra-specific — a *test* for headlines/CTAs). The **"So what?"** test kills any line the customer would shrug at.

**Julian Shapiro's master equation:** `Purchase Rate = Desire − (Labor + Confusion)`. Amplify desire (BAB/PAS), strip labor (fewer fields/steps), strip confusion (plain language, descriptive headlines). cila uses this as the overall content scoring lens.

**Canonical landing-page stack** (5–8 sections, ONE primary CTA repeated, nav stripped): Hero (headline + subhead + CTA + visual) → social-proof bar → Problem (PAS) → Solution/how-it-works (3 steps) → Features→Benefits (FAB) → deeper proof (testimonials w/ metrics, *beside* CTAs) → objections/FAQ → risk reversal (adjacent to CTA) → final CTA → minimal footer. Varies by type: **SaaS** (clarity of what it does; "Start free"), **app** (screenshots + store ratings + install), **ecommerce** (buy box early, reviews above product name, shipping/returns), **lead-gen** (strip to the form), **portfolio** (positioning + work as proof). Benchmarks: all-industry median ~6.6% conversion; 5th–7th-grade reading level converts ~56% better.

**Headlines:** state a clear, specific promise; pass the **5-second clarity test** (stranger knows what + why in 5s). Harry Dry's 3 tests: **can you visualize it? · can you falsify it (numbers/proof)? · could nobody else say it?** Subhead does the clarifying work, not a restatement.

**Social proof / objections / risk reversal:** specific, falsifiable testimonials (numbers > adjectives) placed at decision moments; defuse top 3–5 objections (dedicated FAQ or inline); guarantee/free-trial/"no card" adjacent to the CTA (money-back ≈ +21% sales; risk-reversal language ≈ +32% conversion).

**CTA copy:** one primary action; verb + value, not labor ("Start free" not "Submit"); first-person lifts 10–90% ("Start **my** trial"); 3–4 words; friction microcopy beneath ("No credit card required"). Forms single-column, minimum fields.

Great vs generic: specificity, concreteness/visualizability, customer language, one idea per section, scannability, clarity over cleverness, falsifiability.

Sources: [Shapiro — landing pages](https://www.julian.com/guide/startup/landing-pages) · [Copyhackers — formulas](https://copyhackers.com/2015/10/copywriting-formula/) · [Copyhackers — home page](https://copyhackers.com/2016/06/great-home-page-copy/) · [Harry Dry's 3 rules](https://www.upgrow.io/blog/harry-dry-copywriting-3-rules) · [Marketing Examples](https://marketingexamples.com/) · [CXL — above the fold](https://cxl.com/blog/above-the-fold/) · [Wynter — messaging framework](https://wynter.com/post/website-messaging) · [Unbounce — conversion benchmarks](https://unbounce.com/landing-pages/whats-a-good-conversion-rate/) · [Kissmetrics — CTA best practices](https://www.kissmetrics.io/blog/cta-button-best-practices) · [Replo — landing page anatomy](https://www.replo.app/blog/anatomy-of-a-landing-page) · [Michel Fortin — risk reversal](https://michelfortin.com/copywriting/risk-reversal/)

## 3. Voice, tone & anti-AI-slop (sound human, not generic)

**Voice vs tone:** voice is constant (brand personality); tone flexes by context (hero vs error vs pricing). **NN/g's 4 tone dimensions:** formal↔casual · serious↔funny · respectful↔irreverent · matter-of-fact↔enthusiastic (tone measurably shifts trust/desirability). **Voice Chart** (Podmajersky): product principles → Concepts · Vocabulary (use/ban) · Verbosity · Grammar · Punctuation/Caps. The agent-friendly format is a **"We are X, not Y" + Do's/Don'ts table** — the *not-Y / Don'ts* column is the primary slop guardrail.

**The AI-slop signature** (root cause: regression to the corporate-doc mean — an absence of ideas):
- **Banned words:** unlock · unleash · elevate · empower · supercharge · leverage · seamless · robust · cutting-edge · best-in-class · world-class · game-changer · transformative · innovative · revolutionary · next-level · synergy · holistic · bespoke · curated · delve · turnkey · "solutions" (noun) · ecosystem.
- **Banned phrases:** "in today's fast-paced world" · "in a world where" · "at its core" · "it's worth noting" · "when it comes to" · "look no further" · "the possibilities are endless" · "take your X to the next level" · "we help businesses grow" · "unlock your potential" · "trusted by businesses worldwide".
- **Structural tells:** "not just X, it's Y" · forced rule-of-three · em-dash confetti · "Furthermore/Moreover" openers · meta-commentary ("let's dive in") · hype closers · hedging ("can help / may be able to").
- **Counters:** empty verb → literal action; abstract benefit → number/proof; adjective → show the proof; delete throat-clearing first sentences; state flatly; cut.

**Copy principles (empirical — NN/g):** objective+concise+scannable = **+124%** usability vs promotional "marketese" (hype reads as *less* credible). Specificity over abstraction (Ogilvy's Rolls-Royce clock); **show don't tell** (adjective = claim, number = proof); the **"So what? / Prove it!"** tests (Copyhackers); customer language; ~6th–9th-grade reading level; "omit needless words" (Strunk); read aloud.

**Microcopy (NN/g):** the 3 I's (Inform/Influence/Interact — one job per element); Podmajersky hierarchy **Purposeful → Concise → Conversational → Clear**; buttons = verb + result (never bare "OK"); errors = human, precise, constructive (problem + fix), non-blaming (ban "invalid/illegal"), don't fire too early, preserve input; empty states = status + teach + a pathway.

**Anti-slop edit pass** (separate *editor* role from *writer*): kill-list grep → delete throat-clearing → adjective→proof swap → "So what?" → "Prove it!" → de-formula → em-dash/hedge sweep → every-word-earns-place → voice check → read aloud → reading level → scannability. Constraints get ~80%; the last 20% (authenticity) requires **real extracted specifics** (numbers, customer words, story) — no specifics in, slop out.

## 4. Content IA, narrative, SEO & accessibility (structure & findability)

**StoryBrand SB7:** customer = hero, brand = **guide**. Elements: Character (one desire) · Problem (external/internal/philosophical — buying happens at the *internal*) · Guide (empathy + authority) · Plan (a simple 3-step) · Call to Action (direct + transitional) · avoid Failure (stakes, sparingly) · Success (concrete after-state). The **one-liner**: *[Character] struggles with [Problem]; we provide [Plan] so they can [Success].* **BrandScript = WHAT to say; the wireframe = the ORDER.** Homepage answers, top→bottom: *what do you offer? → how does it make my life better? → what do I do next?* Hero must pass the **3-second grunt test**.

**Information architecture (NN/g):** IA informs the UI; nav is its expression. Tree hierarchy; global nav = top tier (5–7 items); breadcrumbs for depth. **Neither flat nor deep extreme** — choose by content/audience, validate with card sort (grouping) + tree test (findability). Per-type maps: marketing, SaaS, portfolio, docs (Diátaxis intent split). Section order driven by **search intent**.

**Scannability (NN/g):** people scan. **Layer-cake** (heading→heading) is the pattern to *design for* — needs descriptive, distinct subheads; beware **false bottoms**. **F-pattern** is a *symptom of bad formatting* — counteract it. **Inverted pyramid:** conclusion/value first. Front-load every level (headline, first sentence, first words, subheads). Chunk; bullets for parallel items. Concise+scannable+objective = +124% usability.

**Content-first design:** design around real content, never lorem ipsum. **Content hierarchy ≠ visual hierarchy but drives it** — establish content priority first; the visual layer *serves* it (hero = max prominence; junk-drawer footer = min). The **content model** (per page: ordered blocks with type, priority tier, real copy, format) is cila's **content↔design handoff**.

**SEO (quality-first, 2026):** people-first content (who/how/why test); **E-E-A-T** is a lens, not a direct factor (Trust at center); match **search intent** (informational/navigational/commercial/transactional → page type; check what already ranks). On-page: unique descriptive `<title>` + meta description (snippet, not ranking); semantic headings (one H1; order doesn't rank but aids a11y/scan); descriptive anchors (no "click here"); meaningful alt text; internal linking mirrors IA. **JSON-LD** (Organization, BreadcrumbList, Article, Product). **⚠ 2026: FAQ rich results were removed from Google Search (May 7, 2026) — don't target FAQPage schema for SERP features.** **AI Overviews / answer engines = the same people-first fundamentals** — no separate playbook; write self-contained, inverted-pyramid sections with question-style subheads and direct answers up top.

**Accessibility convergence:** plain language / reading level (WCAG 3.1.5); text alternatives / alt as content (1.1.1); descriptive links (2.4.4); semantic heading hierarchy. **Scannability + a11y + SEO converge on the same artifacts** — descriptive headings, descriptive links, meaningful alt, front-loaded plain language — build once, satisfy all three.

## How it builds into cila

- **`CONTENT.md`** — the locked content contract (parallel to `DESIGN.md`): positioning (5 components), job story + ranked pains/gains, ONE core message + headline promise (VoC-grounded), 3–4 pillars (attribute+proof), ICP + awareness stage, objection→reassurance map, **Voice Chart (We are X / not Y + Do/Don't)**, the one-liner, the **IA + section plan** per page type, the cut list, and a VoC swipe file.
- **`content-strategist`** subagent — runs strategy→`CONTENT.md`, collaborating with the user on the *angle* (the content equivalent of design-director deciding the *look*).
- **`copywriting`** skill — frameworks, page anatomy by type, headline/CTA craft, the conversion checklist, the Desire−(Labor+Confusion) lens.
- **`voice`** skill — Voice Chart, the anti-slop kill-list + counters, microcopy, the separate editor pass.
- **`content-structure`** skill — StoryBrand arc, IA, scannability (layer-cake/inverted pyramid), SEO (intent/titles/structured data/2026 flags), a11y convergence, the content-first **content model → visual handoff**.
- **Flow:** content-first — strategy/content (`CONTENT.md`) **before/with** visual direction (`DESIGN.md`), content hierarchy driving visual hierarchy; build writes **real copy** (no lorem); `design-reviewer` also critiques the copy (5-second test, "So what?", slop kill-list, scannability).

The five primitives to enforce as gates: (1) 5-second clarity on the hero, (2) "So what?" per line, (3) Harry Dry's visualize/falsify/differentiate on headlines, (4) the anti-slop kill-list as a hard grep, (5) Desire − (Labor + Confusion) as the scoring lens.
